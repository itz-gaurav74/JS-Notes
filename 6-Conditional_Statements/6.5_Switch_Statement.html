# Switch Statement : Handling Multiple Options - It Is A Control Statement That Is Used To Execute Different Actions Based On Different Conditions. - It Allows Us To Simplyfy The Complex Decision-Making Logic Into A Clear & Easy-To-Read Structure. Syntax: - Case Statement: - Switch Statement Includes A Number Of Case Statements Which Linked To Specific Value. - If The Expression Is Equivalent To That Value, The Code Block Will Be Executed. - Default Statement: - If None Of The Case Statements Match The Expression, Then The Default Statement Is Executed As An Optional Step. - It Is Used To Handle Unexpected Input Or To Provide A Fallback Option. - Break Statement: - It Is Used To Exit The Switch Statement. - Once The Break Statement Is Implimented, The Program Will Be End The Switch Block & Proceed To Execute The Codes That Apper After It. - If The Break Statement Is Not Used, The Program Will Execute The All Blocks. Eg: O/p: Go
