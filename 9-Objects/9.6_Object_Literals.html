# Object Literals: - An Object Literal Is A Comma-Seperated List Of Name-Value Pairs Inside The Curly Braces. # Object Literals Are Using The Following Syntax Rules: - A Colon Seperated Property Name From Value. - A Comma Seperates Each Name/Value Pair From The Next. - There Should Be No Comma After The Last Name/Value Pair. ## It Contsins `key` As Only `string` Or `symbol`: key --> string key --> symbol Example: O/p: Harry Potter Hii Harry Potter Have A Nice Day!
